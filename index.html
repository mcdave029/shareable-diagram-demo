AWS EKS architecture snapshot

Text-only view (HTML/CSS/JS) with no SVG. VPC with public/private subnets, ALB/NLB ingress, managed control plane, node groups (EC2/Fargate), services, CSI storage, CNI, IRSA, and downstream data stores.

Key components

  1. VPC with public & private subnets
  2. Ingress: ALB (HTTP) and optional NLB (TCP/GRPC)
  3. EKS managed control plane
  4. Node groups (EC2 or Fargate), multiple pools
  5. Workloads: web/api pods; jobs/workers
  6. Services: LoadBalancer for web; LB for grpc/tcp
  7. Storage: EBS/FSx/GP3 via CSI
  8. Networking: VPC CNI
  9. IAM Roles for Service Accounts (IRSA)
  10. Data plane attachments: RDS/Aurora, SQS/SNS/Kafka
